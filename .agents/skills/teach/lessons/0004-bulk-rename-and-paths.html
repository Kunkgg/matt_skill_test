Yazi 课程 · 第 3 课

批量改名与拷路径

项目里最常用、ranger/lf 最不顺手的两件事——yazi 都做到了一键。

本课的胜利

选中一批文件，用编辑器一次性把它们全改名；以及把任意文件的路径 / 文件名一键送进剪贴板，粘到别处用。

批量改名：选中 + r

第 2 课里 r 是改单个文件名。当你选中多个文件再按 r，yazi 会打开你的 $EDITOR，把所有名字一列列出来让你改：

  1. 多选若干文件（Space 逐个，或 v 可视区间）。
  2. 按 r —— 编辑器弹出，每行一个文件名。
  3. 改这些名字（行数不能变，只能改内容）。
  4. 保存退出 → yazi 列出「将改哪些」，确认后批量生效。
已为你设置

你的 $EDITOR 之前没设，我已在 .zshrc 设成 vim（你熟 vim 基本操作，正好和 yazi 的 vim 风格一致）。批量改名时编辑器弹出，改完按 Esc 再 :wq 保存退出。想换 nano？把 .zshrc 里 EDITOR=vim 改成 EDITOR=nano 即可。

注意

批量改名时不要增删行——yazi 按行一一对应。改完行数对不上会报错取消，不会乱动你的文件。想删掉某个文件，用 d，不是在编辑器里删行。

对比：ranger 要敲 :bulkrename，lf 得靠插件或手敲命令；yazi 就是「选中 + r」，最直接。

拷路径 / 拷文件名：c 前缀

c 是个「前缀键」——先按 c，再按第二个键决定拷什么（和第 1 课的 gg 同一个套路）：

键拷进剪贴板的内容
c c文件完整路径
c d所在目录路径
c f文件名（含扩展名）
c n文件名（不含扩展名）
记忆钩子

c = copy；第二个字母是内容：complete path（完整路径）、directory（目录）、full name（文件名）、name no ext（去扩展名）。拷的是光标所在项（有选中时拷选中集合）。

WSL 剪贴板要你实测

你的剪贴板走 xsel（已验证可用）。但 xsel 写的是 Linux X 剪贴板——能不能同步到 Windows 剪贴板（让你在 Windows 程序里 Ctrl+V）需要你亲自测：拷一个路径，去 Windows 里粘贴看看。若不通，告诉我，我给你换 wl-clipboard（WSLg 原生，能同步到 Windows）。

预告：; 与 :

还有两个键先记一眼：; 对选中项跑一条 shell 命令（异步），: 同步跑（等它结束）。比如选中几个图片按 ; 输 convert ... 批处理。这属于「自定义工作流」，第 6 课再细讲。

动手验证（沙盒）

跟着做
  1. cd ~/yazi-practice && y。
  2. 用 v 可视模式选中 a.txt b.txt c.txt。
  3. 按 r → vim 弹出三行；改成 a1.txt b1.txt c1.txt，Esc 然后 :wq 存盘退出 → 确认改名。
  4. 光标停在某文件，按 cc 拷完整路径。
  5. 退出 yazi（q），在 shell 里 xsel -bo 看刚拷的内容；再去 Windows 里 Ctrl+V 测同步。
  6. 再试 cf（文件名）、cn（去扩展名）。
检索练习 · 答案都是两键

1. 拷文件完整路径？

2. 拷所在目录路径？

3. 拷文件名（含扩展名）？

4. 拷文件名（不含扩展名）？

看答案

1. cc　2. cd　3. cf　4. cn

批量改名则是另一个键：多选 + r（编辑器批量改）。
