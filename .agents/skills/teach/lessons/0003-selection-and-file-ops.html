Yazi 课程 · 第 2 课

选择与文件操作：复制 / 剪切 / 重命名 / 新建 / 删除

按键大多和 vim 一致；但 ranger / lf 的肌肉记忆有几个危险陷阱，必须先纠偏。

本课的胜利

纯键盘完成一次完整操作：选中几个文件 → 复制/剪切到别处粘贴 → 重命名 → 新建目录 → 安全删除。日常文件管理不再需要鼠标或 cp/mv。

先选，再操作

yazi 和 vim / ranger / lf 一样：先选中目标，再下命令。

键动作
Space选中 / 取消选中当前项
v进入可视模式（区间选择，像 vim 的 v）
V可视模式（反向：取消区间内的选中）
Ctrl+a全选
Ctrl+r反选
Esc清空选择

选中的项左侧会有高亮标记。下面所有「复制/剪切/删除」都作用于当前选中的集合（没选中时作用于光标所在项）。

复制 / 剪切 / 粘贴

键动作
y复制（yank）选中项
x剪切（cut）选中项
p粘贴；遇同名会跳过
P粘贴；遇同名会覆盖
Y 或 X取消当前的复制/剪切状态
⚠ 来自 lf / ranger 的致命陷阱

在 lf 里 d 是「剪切」，在 ranger 里 dd 是「剪切」。但在 yazi 里，d 是「删除（进回收站）」！如果你照 lf/ranger 的习惯按 d 想剪切，文件会被删掉。

记住：yazi 的剪切是 x，不是 d。（钩子：x 像剪刀 ✂ → cut）

vim 用户的注意

vim 里 x 是删一个字符。yazi 里 x 是剪切整个文件——完全不同的语义。y（复制）和 p（粘贴）则和 vim 一致，可放心沿用。

重命名 / 新建 / 删除

键动作
r重命名（rename）
a新建；名字以 / 结尾则建目录
d丢进回收站（可恢复）
D永久删除（不可恢复，慎用）
o / Enter用默认程序打开
O交互式选择用哪个程序打开
小技巧

a 新建时，输入 src/（末尾带斜杠）就直接建目录 src；输入 todo.txt 则建文件。d vs D 的区别就是「回收站」vs「永久删除」——养成默认用 d 的习惯，只有确定不要了才 D。

ranger ↔ lf ↔ yazi 对照

动作rangerlfyazi
复制yyyy
剪切dddx
粘贴pppp
重命名cwcr
删除dDDd/D

规律：ranger 是「双键」、lf 是「单键但 d=剪切」，yazi 是「单键且 x=剪切 / d=删除」。从 lf 迁移最省力，从 ranger 迁移要改掉双键习惯。

动手验证（在沙盒里练，安全）

我已经在 ~/yazi-practice/ 建了几个假文件（a.txt b.txt c.txt notes.md + sub/）。在这里随便折腾：

跟着做
  1. y 启动，导航进 ~/yazi-practice（或先 cd ~/yazi-practice && y）。
  2. 光标停在 a.txt，按 Space；移到 b.txt，再 Space —— 两个都被选中。
  3. 按 y 复制；进 sub/（l），按 p 粘贴 → sub 里多了两份。
  4. 退回上级，光标到 c.txt，按 r 改名为 c.md。
  5. 按 a，输入 draft/ 回车 → 新建目录 draft。
  6. 选中任意一份副本，按 d → 进回收站（列表里消失）。想确认能恢复，去 ~/.local/share/Trash 看。
  7. Esc 清空选择。
检索练习 · 先想再点开（答案都是单键）

1. 复制（yank）选中文件？

2. 剪切（cut）选中文件？

3. 粘贴？

4. 丢进回收站？

5. 重命名？

6. 新建（末尾 / 建目录）？

看答案

1. y　2. x　3. p　4. d　5. r　6. a

两个钩子：x 像剪刀=cut；d=delete=回收站（lf 用户尤其别把 d 当剪切）。
