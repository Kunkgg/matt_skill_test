Yazi 课程 · 第 4 课

多 tab 与查找

两个让「快」真正落地的能力——并排开几个目录，以及在任意深度一秒定位文件。

本课的胜利

开多个 tab 并行处理不同目录，还能跨 tab 复制/移动文件；以及分清过滤 / 查找 / 搜索三种「找文件」的方式，按场合挑最快的那个，不再靠肉眼滚列表。

多 tab：并排打开几个目录

一个 tab 就是一个独立的目录视图——各自导航、互不打扰。你不必为了在两个目录间搬东西而反复 cd 进出。

键动作
tt在当前目录新建一个 tab
1-9切到第 N 个 tab
[ / ]上一个 / 下一个 tab
{ / }与上一个 / 下一个 tab 交换位置
Ctrl+c关闭当前 tab
记忆钩子

t 是 tab 前缀（和你已经熟的 g、c 前缀一个套路），所以新建是 tt；数字 1-9 直达；[] 翻动。新 tab 继承你当前所在目录，不是 home。

别按错退出键

关掉一个 tab 是 Ctrl+c。q 是退出整个 yazi（配合 y wrapper 还会 cd 到当前目录）。关单个 tab 别按成 q。

跨 tab 复制 / 移动——tab 的核心价值

yazi 的剪贴板（yank / cut）是全局的，不局限在一个 tab 里。所以最常见的玩法是：

  1. tab A：导航到源目录，Space 选中文件，y（拷）或 x（剪）。
  2. tt 开新 tab，l 进目标目录。
  3. p 粘贴——文件就过来了。

第 2 课学的 y/x/p 原封不动，只是换了张 tab。这也是 ranger/lf 用户最容易忽略的提速点。

找文件的三种姿势（本课重点）

yazi 有三个看起来很像、但范围完全不同的「找」。搞清区别，你就知道每个场合该用哪个，不用再滚长列表：

键干什么范围
f边打字边收窄可见列表（隐藏不匹配项）当前目录
/ ?移动光标到下一个 / 上一个匹配名当前目录
n N在已找到的匹配间向后 / 向前翻（配合 /）
s用 fd 按文件名递归搜整棵子树
S用 ripgrep 按文件内容递归搜整棵子树

什么时候用哪个

  • 只想看当前目录里某一类（比如所有 .txt）→ f，给当前视图加个临时筛子，Esc 清除。
  • 当前目录文件多，想跳到某个名字（如 report 开头）→ / 输入，n/N 在匹配间翻——就是 vim 里那个 /，不隐藏任何文件，只挪光标。
  • 不知道文件埋在哪层子目录，按名字找 → s（fd 递归）。
  • 要找「哪个文件里写了 XXX」→ S（ripgrep 搜内容）。
记忆钩子

f = filter（收窄当前列表）；/ 就是你 vim 里那个 /（当前列表里跳光标）；小写 s 搜名字、大写 S 搜内容——大写 = 更「深」，钻进文件里面去。s/S 跑起来时按 Ctrl+s 可取消。

✅ 实测：s 在本机开箱可用

你这台 Ubuntu 把 fd 装成了 fdfind、fd 只是个 shell 别名——但 yazi 在找不到 fd 时会自动回退到 fdfind（已从 yazi 二进制确认），所以 s 正常工作，无需手动建软链。S 用的 rg 是真二进制（/usr/bin/rg），也没问题。跨平台提醒：换到一台既没 fd 也没 fdfind 的机器（比如干净容器），s 才会失效——届时装个 fd 即可。

动手验证（沙盒）

跟着做
  1. cd ~/yazi-practice && y。
  2. tab：tt 开新 tab → 1 回第一个 → ] 切下一个 → Ctrl+c 关掉当前的。
  3. 跨 tab 复制：tab 1 选中 a1.txt（Space），y 拷；tt 新 tab，l 进 sub，p 粘贴 → sub 里出现副本。Ctrl+c 关掉这个 tab。
  4. 过滤：回根目录，按 f 打 txt → 只剩 a1.txt b1.txt（sub 里的也算）；Esc 清除，列表复原。
  5. 查找：按 / 输 c → 光标跳到 c.md；Esc 退出。
  6. 搜名字：按 s 输 report → 应能在结果里挑出 sub/buried_report.md（它不在当前视图里，证明是递归搜）。若报 fd 错误，记下来反馈我。
  7. 搜内容：按 S 输 quarterly → ripgrep 找到 sub/buried_report.md（按内容命中）。
检索练习 · 答案都是一键

1. 只想给当前目录加个临时筛子、边打字边收窄可见列表？

2. 当前目录里，vim 风格把光标跳到下一个匹配的文件名？

3. 不知道埋在哪层，递归整棵子树按文件名搜？

4. 递归整棵子树，按文件内容搜？

看答案

1. f（filter）　2. /（find，n/N 翻）　3. s（fd 按名）　4. S（rg 按内容）

tab 那组速记：新建 tt · 切换 1-9 / [] · 关闭 Ctrl+c。
